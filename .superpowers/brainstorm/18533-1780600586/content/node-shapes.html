Node Shape Language

Each node in the cycle should have a distinct 3D shape that communicates its role. Which direction feels right for representing these system components?

A: Iconic Wireframes

Each node is a distinct wireframe shape that symbolizes its role: terminal prompt (trigger), hexagonal hub (orchestrate), code brackets (coding), shield (validator), bullseye (resolution). Clean, minimal, instantly readable.

B: Geometric Primitives

Each node is a different 3D wireframe primitive: cube (input), octahedron (orchestrate), cylinder (coding), geodesic sphere (validator), star (resolution). More abstract, but the shapes themselves carry meaning through convention (octahedron = hub/router, cylinder = process).

C: Mini Dashboard Cards

Each node is a small 3D floating card/panel with a header bar, monospace labels, and contextual micro-content (code snippets, progress bars, status dots). Like miniature dashboard widgets floating in 3D space. Most literal representation.
