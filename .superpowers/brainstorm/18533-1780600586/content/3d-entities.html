3D Entity Style

Simple 3D shapes with surface detail — closer to the original aesthetic but semantically meaningful. Each entity is a recognizable object viewed mostly head-on, with detail "traced" on the front face.

Solid 3D Boxes with Front-Face Detail

CyberClaw and Dev VM are solid 3D boxes (like actual servers/machines) with subtle isometric depth. All the detail lives on the front face — status indicators, nested boundaries, sub-process stages. The surface material matches your existing cream/gold palette. In Three.js: BoxGeometry with MeshPhysicalMaterial, detail rendered via HTML overlays or texture. SSH connection is a dashed line with travelling particle dots.

How this maps to Three.js

CyberClaw box: BoxGeometry + MeshPhysicalMaterial (same clearcoat/roughness as current nodes). Front face gets a Threlte <HTML> overlay with monospace labels, or a canvas texture. Slightly smaller than VM box.
Dev VM box: Larger BoxGeometry. Front face shows nested boundaries (automate_loop, stages) via HTML overlay. Subtle depth with edge lines.
SSH connection: Same TubeGeometry approach as current arcs, but with animated particles travelling along it (already implemented in StepAnimations).
Active highlighting: Same spring-based emissive glow from current DiagramNode, applied to whichever entity is active for the current scroll step.
Scroll steps map to: Step 0 highlights CyberClaw + SSH, Step 1 highlights CyberClaw orchestrator, Step 2 highlights VM + automate_loop stages, Step 3 highlights validator, Step 4 highlights success badge.
