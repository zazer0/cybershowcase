Entity-Based Diagram Layout

Representing the actual infrastructure: CyberClaw machine, SSH tunnel, Dev VM, and automate_loop within the VM. Two spatial arrangements.

A: Left-to-Right Infrastructure

CyberClaw on left, SSH tunnel across, Dev VM on right (larger, containing automate_loop and its 3-stage sub-process). The VM boundary clearly shows containment. Feedback loop arrows show the retry cycle. In 3D, the VM could have depth (like an open server rack) with automate_loop stages arranged inside.

B: Top-Down Command Chain

CyberClaw at top (commanding), SSH drops down into a wide Dev VM container below. automate_loop's 3 stages spread horizontally inside the VM. Feedback loops back up to CyberClaw. In 3D, the VM could recede in depth (like looking into a server), with CyberClaw hovering above. Natural reading order matches the command hierarchy.

These are 2D mockups of the spatial layout. In 3D, each entity gets depth, wireframe edges, and the scroll-driven highlighting from the current implementation.
